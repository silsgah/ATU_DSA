📖 Course description

BCP 201 expands students' programming skills and provides deep understanding of: data abstraction & ADTs, algorithm design (greedy, divide & conquer, dynamic programming, randomized, backtracking), runtime & memory analysis, and implementation of generic collections using high-level languages.

🎯 Course objectives

  • Understand concepts of data structures and abstract data types
  • Apply algorithm analysis to determine runtime & memory usage
  • Use design techniques: greedy, divide-and-conquer, DP, randomized, backtracking
  • Implement generic collections: linked lists, stacks, queues
  • Write and reason about recursive methods

📚 Mode of delivery & assessment

  • 📖 Classroom lectures — concept clarity & worked examples
  • 🏠 Take-home assignments — algorithmic thinking & implementation
  • 💻 Practical implementation — coding ADTs in high-level language
✔️ Final project · problem-solving focus

🗂️ Course content overview

01 · Intro to Data Structures

Organizing & storing data efficiently, classification: linear (arrays, linked lists) & non-linear (trees, graphs), static vs dynamic.

02 · Abstract Data Types (ADTs)

Mathematical model by behavior: List, Stack, Queue, Map, Set, Tree, Graph. Interface vs implementation – modularity & abstraction.

03 · Algorithms & Complexity

Big-O classes, analysis of runtime/memory. Design paradigms: greedy, divide & conquer, DP, randomized, backtracking.

04 · Lists

ArrayList (contiguous) vs Singly/Doubly Linked Lists. Operations, tradeoffs, cache friendliness, dynamic sizing.

05 · Stacks & Queues

LIFO / FIFO. push/pop, enqueue/dequeue. Applications: function calls, scheduling, BFS, undo/redo.

06 · Recursion

Base case + recursive case. Factorial, Fibonacci, Tower of Hanoi, tree traversals, merge sort.

07 · Searching & Hashing

Linear O(n), Binary O(log n), Hashing O(1) average, collision handling (chaining, probing). Dictionaries & caches.

08 · Trees

Binary Trees, BST, AVL, traversals (inorder, pre, post). Balanced trees, heaps, tries.

09 · Graphs

Vertices, edges, directed/undirected, weighted. BFS, DFS, Dijkstra, Topological sort, MST (Kruskal/Prim).

⚡ Complexity & algorithm design

📈 Big-O complexity classes

O(1) Constant
O(log n) Logarithmic
O(n) Linear
O(n log n) Linearithmic
O(n²) Quadratic
O(2ⁿ) Exponential

⏱️ Analyze worst-case / average-case, space complexity matters.

🧠 Algorithm design techniques

  • Greedy – locally optimal (Dijkstra, Huffman)
  • Divide & Conquer – split & merge (Merge Sort, Quick Sort)
  • Dynamic Programming – memoize subproblems (Fibonacci, Knapsack)
  • Randomized – improve average performance (Quicksort random pivot)
  • Backtracking – explore & prune (N-Queens, Sudoku)

📌 Abstract Data Types & classic structures

📋 List ADT

ArrayList: O(1) access, O(n) insert/delete middle.
Singly Linked List: O(1) insert at head, O(n) search.
Doubly Linked List: next/prev pointers, efficient at both ends.

🥞 Stack & Queue (ADT)

Stack (LIFO): push, pop, peek — recursion, undo/redo.
Queue (FIFO): enqueue, dequeue — scheduling, BFS, print queue.

🌳 Trees & Hashing

Binary Search Tree (O(h)), AVL self-balancing (O(log n)), Hash Table (average O(1)) — collision: chaining / open addressing.

🕸️ Graph algorithms

BFS (queue), DFS (stack/recursion), Dijkstra (shortest path), MST (Prim/Kruskal), topological sort for DAGs.

🔄 Recursion in depth

📌 Base & recursive case

def factorial(n):
    if n == 0:          # base case
        return 1
    return n * factorial(n-1)  # recursive case

Every recursive function must have a base case to avoid infinite loops.

🔍 Searching & Hashing

Linear search O(n)

Works on unsorted data, iterate through each element.

Binary search O(log n)

Requires sorted array, divide in half repeatedly.

Interpolation search O(log log n)

Estimate position based on value distribution, for uniformly distributed data.

Hashing O(1) avg

Hash function → index. Collision resolution: chaining (linked lists) or open addressing (linear probing). Load factor = n/m.
