\section{Assignment 1: Arrays (45 Marks)}
\subsection*{Topic: University Course Registration Analytics}

\subsubsection*{Background}
Accra Technical University stores student registrations for a specific course in a Python list representing the sequence of entries as they arrive.
\begin{lstlisting}[language=Python]
registrations = [
    1023, 1050, 1023, 1102,
    1050, 1023, 1201, 1102,
    1300, 1023
]
\end{lstlisting}
Each integer in the list represents a unique Student ID.

\subsubsection*{Problem Statement}
Write a Python program that implements efficient algorithms to perform the following operations:
\begin{enumerate}
    \item \textbf{Most Frequent Registrant (8 Marks):} Find the student ID that appears the most times in the registration list.
    \item \textbf{Ordered Deduplication (10 Marks):} Remove all duplicate student IDs from the list while preserving the exact order of their first appearance.
    \item \textbf{First Unique Student (9 Marks):} Identify the first student ID in the registration list that appears exactly once.
    \item \textbf{Contiguous Subarray Sum (10 Marks):} Determine whether there exists a contiguous subarray of registrations whose sum equals a given target value $T$ (e.g., check if a sequence of IDs sum to a target).
    \item \textbf{Complexity Analysis (8 Marks):} Analyze and state the worst-case time and space complexities for each of your four solutions using Big-O notation.
\end{enumerate}

\subsubsection*{Expected Output Format}
Your script should print the results in the following format:
\begin{lstlisting}[backgroundcolor=\color{black!5!white}]
Most frequent student: 1023
Unique registrations: [1023, 1050, 1102, 1201, 1300]
First non-repeated student: 1201
Subarray with target sum exists: True
\end{lstlisting}

\section{Assignment 2: Linked Lists (45 Marks)}
\subsection*{Topic: University Helpdesk Ticket Queue}

\subsubsection*{Background}
The university IT helpdesk maintains support tickets in the order they arrive. To support dynamic additions and updates without the memory overhead of contiguous array shifts, the department wants to use a Singly Linked List. A ticket node contains:
\begin{itemize}
    \item \texttt{ticket\_id} (Integer)
    \item \texttt{student\_name} (String)
    \item \texttt{issue} (String)
\end{itemize}

\subsubsection*{Problem Statement}
Implement a singly linked list class (\texttt{TicketQueue}) from scratch that supports the following operations:
\begin{enumerate}
    \item \textbf{Enqueue Ticket (8 Marks):} Add a new ticket to the end of the queue.
    \item \textbf{Priority Insert (10 Marks):} Insert a new priority ticket immediately after a specified ticket ID.
    \item \textbf{Resolve Ticket (9 Marks):} Delete a resolved ticket from the queue given its ticket ID.
    \item \textbf{Find Middle Ticket (8 Marks):} Find the middle ticket of the queue in a single traversal pass (using the fast/slow pointer method).
    \item \textbf{Reverse Queue (10 Marks):} Reverse the queue in-place (manipulating pointers without copying data).
    \item \textbf{Display Queue (4 Marks):} Traversal method to print all ticket IDs and details.
    \item \textbf{Complexity Analysis (6 Marks):} State the worst-case time complexity of each operation.
\end{enumerate}

\subsubsection*{Starter Classes}
Use the following starter code to begin your implementation:
\begin{lstlisting}[language=Python]

        self.ticket_id = ticket_id

        self.issue = issue
        self.next = None

        self.head = None
