Concept

What Is a Binary Heap?

A Binary Heap is a special type of complete binary tree that is designed to answer one specific question very efficiently:

“What is the absolute largest (or smallest) element in this dataset?”

Structural Requirements

  • It is a complete binary tree: every level is fully filled except possibly the last, which is filled from left to right.
  • It enforces a strict ordering rule between each parent and its children.
  • There is no guaranteed relationship between sibling nodes — unlike a Binary Search Tree.

Time Complexity Summary

OperationTime
Peek minimum or maximumO(1)
Insert an elementO(log N)
Remove minimum or maximumO(log N)
Build heap from N items (Floyd)O(N)

Core Rule

The Heap Property

Min-Heap

Every parent node holds a value less than or equal to both of its children.

The root is always the smallest element in the entire structure.

Max-Heap

Every parent node holds a value greater than or equal to both of its children.

The root is always the largest element in the entire structure.

Key difference from Binary Search Trees: In a heap, siblings have no defined ordering relative to each other. A left child is not necessarily smaller than a right child.

Storing a Heap in an Array

Because a heap is always a complete binary tree, its nodes have no gaps. We can store the entire tree level by level in a one-dimensional array — no pointers or node objects are needed.

Index Arithmetic (0-indexed array)

For any element at index i:

Parent: (i − 1) // 2
Left child: 2 × i + 1
Right child: 2 × i + 2
These three formulas let us navigate the tree structure without any pointers. The root is always at index 0.

Core Operation

Sift-Up — Restoring the Heap After Insertion

When a new element is inserted, it is appended to the end of the array. It may be smaller than its parent, which would violate the heap property. Sift-Up corrects this by moving the element upward.

Algorithm Steps

1
Append the new value to the end of the array.
2
Find its parent using index (i − 1) // 2.
3
If the new element is smaller than its parent (for a Min-Heap), swap them.
4
Repeat from the new position until the element is larger than its parent, or it reaches the root.
In the worst case, the element travels from the bottom level to the root: at most log N swaps.
Time complexity: O(log N)

Python Code

def _sift_up(self, index):
    # While we have a parent node AND
    # the parent is larger than the current element:
    while self.has_parent(index):
        parent_i = self.get_parent_index(index)

        if self.heap[parent_i] > self.heap[index]:
            # Swap with the parent
            self.heap[parent_i], self.heap[index] = (
                self.heap[index], self.heap[parent_i]
            )
            index = parent_i   # move focus upward
        else:
            break  # heap property is satisfied

Core Operation

Sift-Down — Restoring the Heap After Extraction

When the root is removed (the minimum in a Min-Heap), the last element in the array is placed at the root. It is likely too large and must sink down to its correct position.

Algorithm Steps

1
Record the root value as the result. Move the last element to index 0. Remove the last position.
2
Compare the element with its left and right children.
3
Identify the smaller of the two children (for a Min-Heap).
4
If the element is larger than the smaller child, swap them.
5
Repeat until the element is smaller than both children, or it reaches a leaf node.
Time complexity: O(log N)

Python Code

def _sift_down(self, index):
    # Continue while there is at least a left child
    while self.has_left_child(index):
        left  = self.get_left_child_index(index)
        right = self.get_right_child_index(index)

        # Assume left child is the smaller candidate
        smaller = left

        # If right child exists and is smaller, use it
        if (self.has_right_child(index) and
                self.heap[right] < self.heap[left]):
            smaller = right

        # If current element is already smaller, stop
        if self.heap[index] < self.heap[smaller]:
            break

        # Swap and continue downward
        self.heap[index], self.heap[smaller] = (
            self.heap[smaller], self.heap[index]
        )
        index = smaller

Abstract Data Structure

Priority Queues

A Priority Queue is an abstract data structure: it defines a contract that elements leave in order of priority, not in order of arrival. A Binary Heap is the standard data structure used to implement it efficiently.

Comparison: Regular Queue vs. Priority Queue

Regular Queue (FIFO)Priority Queue
First inserted leaves first.Highest priority leaves first.
No concept of importance.Each element carries a priority.
Implemented with an array or linked list.Implemented with a Binary Heap.
Dequeue: O(1)Dequeue: O(log N)

Real-World Applications

Operating System Scheduling

The CPU always runs the highest-priority process first. New high-priority processes can displace lower-priority ones.

Network Packet Routing

Routers use priority queues to forward urgent packets (e.g., voice calls, video streams) before bulk data.

Dijkstra’s Shortest Path Algorithm

A Min-Heap always selects the nearest unvisited node first — fundamental to GPS navigation and routing software.

Python Standard Library

Python’s heapq Module

Python provides a built-in heapq module that implements a Min-Heap directly on top of a regular Python list. No separate class is required.

Core Functions

FunctionWhat it doesTime
heappush(h, x)Insert x and restore heapO(log N)
heappop(h)Remove and return smallestO(log N)
heapify(h)Convert list to heap in-placeO(N)
h[0]Read smallest without removingO(1)
Important: Python’s heapq only provides a Min-Heap. To simulate a Max-Heap, insert the negated value and negate the result on extraction.

Usage Example

import heapq

numbers = [5, 1, 9, 3, 2]

# Build a Min-Heap in place — O(N)
heapq.heapify(numbers)
# numbers is now: [1, 2, 9, 3, 5]

# Read the minimum without removing it
minimum = numbers[0]   # -> 1

# Remove and return the minimum — O(log N)
val = heapq.heappop(numbers)   # -> 1

# Insert a new element — O(log N)
heapq.heappush(numbers, 4)

# ---- Simulating a Max-Heap ----
max_heap = []
for x in [5, 1, 9]:
    heapq.heappush(max_heap, -x)
# Remove the largest element:
largest = -heapq.heappop(max_heap)   # -> 9

Classic Problem

Finding the K-th Largest Element

Problem: Given an unsorted list of N numbers, find the K-th largest element.
Constraint: The solution must run in O(N log K) time, not O(N log N).

Why Not Just Sort?

Sorting takes O(N log N) and requires holding all N elements in memory. For very large datasets (hundreds of millions of records), this is too slow and uses too much memory.

Strategy Using a Min-Heap

1
Maintain a Min-Heap capped at size K.
2
For each element in the list, push it onto the heap.
3
If the heap exceeds size K, remove the smallest element (the root).
4
After all elements are processed, the root is the K-th largest overall.
At the end, the heap contains exactly the K largest elements seen. Its minimum (the root) is the K-th largest.

Python Solution

import heapq

def find_kth_largest(nums: list, k: int) -> int:
    min_heap = []

    for num in nums:
        heapq.heappush(min_heap, num)

        # If heap grows past K, discard the smallest.
        # This keeps only the K largest elements.
        if len(min_heap) > k:
            heapq.heappop(min_heap)

    # The root is the K-th largest.
    return min_heap[0]


# Example
nums = [3, 2, 1, 5, 6, 4]
k = 2
print(find_kth_largest(nums, k))   # -> 5
Time: O(N log K)
Space: O(K) — heap holds at most K elements

Advanced Topic

Floyd’s O(N) Heap Construction

There are two approaches to building a valid heap from an unsorted array of N elements.

Approach 1 — Insert One by One

Call push() (which calls Sift-Up) N times — once per element.

Each insertion: O(log N)
Total: O(N log N)
This is the straightforward but slower approach.

Approach 2 — Floyd’s Bottom-Up Method

Start from the last non-leaf node (index (N // 2) − 1) and apply Sift-Down to every node, moving upward toward the root.

Total: O(N) ← strictly faster 
n = len(arr)
# From last non-leaf to root:
for i in range(n // 2 - 1, -1, -1):
    sift_down(arr, i, n)
Python’s heapq.heapify() uses exactly this method.
Why is Floyd’s method O(N)? Approximately half of the nodes in a complete binary tree are leaves that require no sifting. Only a small number of nodes exist near the root where the most work is done. When the total work across all levels is summed, it converges to O(N) — proven using geometric series analysis.
